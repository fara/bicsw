Perfiles de usuarios
Arriba  Abajo  Siguiente

El sistema BIC esta pensado para distintos perfiles de usuarios que se detallan a continuación:

   
·   Arquitecto  
·      Administrador  
·      Reportero  


   Arquitecto:

El arquitecto es el encargado de configurar todos los datos necesarios para que desde un proyecto en particular se pueda generar de manera adecuada un reporte. En definitiva el arquitecto es el único que tiene acceso a configurar los hechos, tablas y atributos. El mismo también puede configurar las , filtros y .

Por lo cual el arquitecto puede visualizar el total de las pantallas menos la de administración de usuarios, del cual se ocupa el administrador.

   Administrador:

El administrador es el encargado de gestionar a todos los usuarios del sistema. Inicialmente por defecto el sistema crea dos usuarios, uno con perfil de administrador y otro con perfil de arquitecto. Luego el administrador dispone de la posibilidad de crear distintos usuarios bajo los tres perfiles aceptados por el sistema (Arquitecto, Adminitrador, Reportero).

Este usuario solo puede visualizar la pantalla de administración de usuario.

   Reportero:

El reportero debe ser creado inicialmente por el administrador y es el responsable de gestionar todos los reportes de los proyectos a los que esta asignado. Con gestionar reportes nos referimos a que le permite crear, configurar, generar o borrar los mismos. Asimismo el reportero puede crear, configurar o borrar tanto como métricas.

Este usuario únicamente puede visualizar la pantalla donde se encuentran los reportes, y de cada proyecto.
